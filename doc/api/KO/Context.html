KO::Context

Context defines a system “state”. A context might specify a set requirments, database fixtures, objects, mocks, or file system setups —any presets that need to be in place for a feature and/or scenario to operate. 

  context "Instance of Dummy String" do

    setup do
      @string = "dummy"
    end

  end

NOTE: Some context cannot be fully isolated. For instance, once a library is loaded it cannot be unloaded.
