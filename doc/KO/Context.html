Overview

Context defines a system “state”. A context might specify a set of requirments, database fixtures, objects, mocks, or file system setups –any presets that need to be in place for a feature and/or scenario to operate.

KO.context "Instance of Dummy String" do

  before :all do
    @string = "dummy"
  end

end

NOTE: Some context cannot be fully isolated. For instance, once a library is loaded it cannot be unloaded.

def after(type=:each, &block)
  raise ArgumentError, "invalid after-type #{type}" unless [:each, :all].include?(type)
  type_method = "after_#{type}"
  remove_method(type_method) rescue nil #if method_defined?(:teardown)
  define_method(type_method, &block)
end
